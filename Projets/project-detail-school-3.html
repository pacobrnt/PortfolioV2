Forum cybersécurité

Intervenant au lycée Gabriel Fauré ainsi qu'au collège de Groisy

Le Forum Cybersécurité du lycée Gabriel Fauré, lancé en 2024 par les enseignants du BTS SIO, vise à sensibiliser élèves et personnels aux menaces numériques à travers des ateliers interactifs. J’y ai participé en 2025 en tant qu’intervenant sur un atelier consacré aux dangers des PDF piégés. L’édition 2025 a rassemblé plus de 700 participants et mobilisé plus de 50 étudiants et enseignants.

Objectifs de l'atelier

  • Sensibiliser aux fichiers courants pouvant contenir des menaces
  • Développer une vigilance face aux pièces jointes
  • Expliquer les techniques d’exploitation via PDF
  • Apprendre à analyser un fichier suspect
  • Montrer les conséquences concrètes d’une attaque
  • Adopter les bons réflexes de sécurité numérique
  • Contribuer à une culture numérique responsable
  • Encourager l’intérêt pour l’analyse de fichiers et la cybersécurité

Détail de l'atelier de l'atelier

L’atelier portait sur les virus transmis par PDF. Pour réaliser cela, nous disposions de deux configurations : celle du groupe Yanis, Louai et Paco nommée “Virus”, et celle des participants nommée “Utilisateurs”.

Sur le PC “Virus”, à l’aide de commandes trouvées grâce à un TP destiné aux deuxièmes années fourni par M. Naville, nous avons infecté un fichier PDF de notre choix.

Le groupe “Utilisateurs” était mis devant un site proposant trois formations pour devenir riche grâce au e-commerce (e-books) sous forme de PDF. Une fois le virus installé sur la machine “Utilisateur”, nous leur présentions les différentes fonctionnalités accessibles avec le virus (Keylogger, Webcam, diffusion de leur écran).

Leur objectif était d’identifier quelle formation était infectée. Nous avons ensuite échangé avec eux sur les méthodes pour éviter les virus et se protéger : mettre à jour ses logiciels pour bénéficier des derniers antivirus, faire preuve de prudence avec les mails, et rester vigilant sur les sites visités.

Schéma de l'atelier

Configuration du virus avec Kali Linux

Tout d'abord, sur la machine avec l’OS Kali Linux nous avons lancé la console de Metasploit en utilisant la commande msfconsole.

Ensuite, nous avons recherché un exploit spécifique ciblant les fichiers PDF Adobe sur les systèmes Windows en utilisant la commande search type:exploit platform:windows adobe pdf.

Après avoir sélectionné l'exploit à utiliser pour notre attaque via fichier PDF, nous avons exécuté la commande suivante dans Metasploit : exploit/windows/fileformat/adobe_pdf_embedded_exe. Cette commande permet de générer un fichier PDF malveillant qui intègre un exécutable.

Ensuite, le fichier PDF généré a été déplacé sur notre site Web d’ebooks piégés, accessible par les participants de l’atelier.

Une fois le virus créé et intégré au site, nous avons configuré le payload pour établir une connexion inversée via Meterpreter, en utilisant le module multi/handler : ces paramètres définissent l’adresse IP de l’attaquant (LHOST) et le port utilisé pour écouter la connexion entrante depuis la cible (LPORT).

Après cette configuration, nous avons lancé l’écoute avec la commande. Le terminal indique alors que Metasploit est en attente de la connexion depuis la cible. Une fois que la victime ouvre le fichier PDF piégé, la session Meterpreter est automatiquement établie, comme visible dans la capture.

Scénario de l’atelier Virus

L'atelier visait à montrer comment un fichier PDF infecté peut être utilisé pour une attaque. Pour cela, nous avons utilisé un ordinateur tournant sous Kali Linux, avec l'adresse IP 192.168.10.200. Notre objectif principal était de créer un fichier PDF intentionnellement infecté, exploitant une faille spécifique dans la version d'Adobe Reader 8.1.1, ce qui permettrait de faire fonctionner un logiciel malveillant à distance.

Voici le lien de téléchargement de la version d’Adobe 8.1.1 possédant une faille de sécurité : http://www.oldversion.com/windows/acrobat-reader-8-1-1

Trois fichiers PDF ont été créés, dont un seul a été intentionnellement infecté. Les participants ont été invités à télécharger ces fichiers sur leurs machines Windows en accédant directement à la page “Nos Ebooks” depuis le site codé en HTML.

Les participants ont été invités à télécharger les 3 ebooks tout en ne sachant pas lequel des 3 est infecté. Une fois les 3 ebooks téléchargés,

Ils ont été invités à sélectionner les 3 ebooks en même temps pour les ouvrir simultanément. Une fois les 3 fichiers PDF ouverts avec la version 8.1.1 d’Adobe Reader, plusieurs messages apparaissent successivement pour confirmer l’ouverture du fichier infecté. Il faut notamment enregistrer le fichier en remplaçant l’existant, puis cliquer sur "Open" pour lancer le virus. Ces étapes ne font que valider l’exécution du malware. Lorsqu’ebook1.pdf infecté est ouvert, notre machine Kali reçoit un message de Meterpreter, indiquant que la machine cible est compromise et qu’un accès distant est établi. Ce message affiche notamment l’adresse IP de l’attaquant et de la cible, ainsi que la date et l’heure du déclenchement. Meterpreter, outil du framework Metasploit, permet à l’attaquant de contrôler à distance le système compromis, d’exécuter des commandes, d’extraire des données et d’explorer le réseau.

Moyens de prévention

Pour éviter ce type de problème, on peut utiliser des logiciels comme Windows Defender, le programme antivirus gratuit de Windows. Mais, pour la démonstration, on avait désactivé Windows Defender. Si on l'avait laissé activé, le PDF infecté aurait été bloqué dès son ouverture. Pour éviter une attaque via un PDF piégé, il est essentiel d’adopter plusieurs bonnes pratiques de sécurité. Il faut tout d’abord éviter d’ouvrir des fichiers PDF provenant de sources inconnues ou douteuses, notamment par mail ou via des sites non sécurisés. L’utilisation d’un lecteur PDF sécurisé, comme Sumatra PDF, ou l’ouverture des fichiers dans un environnement isolé (machine virtuelle, sandbox) permet de limiter les risques. Il est également recommandé de désactiver JavaScript dans les lecteurs PDF et de scanner les fichiers avec un antivirus à jour avant de les ouvrir. Des outils d’analyse comme VirusTotal ou PDF Examiner peuvent aussi être utilisés pour détecter d’éventuels scripts malveillants. Enfin, il convient de désactiver les fonctions d’ouverture automatique des pièces jointes et de restreindre l’exécution de macros ou d’actions automatiques dans les lecteurs PDF.
